# Trash（垃圾筒）UI & 交互改进

> 对应 `Views/TrashView.axaml`（296 行）和 `ViewModels/TrashViewModel.cs`（565 行）。

## 文件卡片

### 无多选
只能单张恢复或清理，无法批量操作。

**方案**: 垃圾筒加多选模式：
- 长按/右键进入多选，卡片上显示复选框
- 工具栏出现"全选 / 取消全选 / 批量恢复 / 批量清理"
- 仅云端和仅本地各自独立多选

### 卡片信息太少
每张卡片只显示文件名，不知道什么时候删的、文件多大。

**方案**: 卡片上追加：
- 删除日期（如 `2024-07-09 删除`）
- 文件大小（如 `12.3 MB`）
- 如果是云端已上传的，标记原始路径

### 两张卡片布局几乎一样
"已在云端"和"仅本地"两段除了按钮数量外视觉高度相似，区分度不够。

**方案**:
- "已在云端"段：卡片加青蓝色顶部细条
- "仅本地"段：卡片加灰色顶部细条
- 两段间距加大到 32px

## 操作反馈

### 清理无撤销
`CleanSingleCommand` 直接物理删除（云端 + 本地），无后悔机会。

**方案**: 清理本地文件后，Toast 提示"已清理 xxx"，保留"撤销"按钮 5 秒。5 秒内点击撤销恢复本地文件（重新创建 `.trashed` 软删除标记）。云端清理因涉及 OSS API 无法可靠撤销，仅给出确认对话框。

### Toast 挡住操作区
`ToastMessage` 在 Grid.Row="1" 底部居中，可能盖住最下面一行的文件卡片。

**方案**: Toast 位置改为顶部横幅（在标题栏和卡片区之间），不遮挡内容。

### "清空垃圾筒"无二次确认
点击直接弹出三选对话框（清理云端 / 仅本地 / 取消），但没有前置的"共有 N 个文件"提示。

**方案**: 对话框标题中加入统计："共有 12 个云端文件、8 个本地文件，确认清空？"

## 导航

### 无返回原位提示
在垃圾筒执行"恢复"后，文件回到媒体库，但用户需要手动回到 Gallery 页查找。

**方案**: 恢复成功后 Toast 中加"跳转到文件"链接，点击切换到 Gallery 并自动滚动/定位到该文件。

### 无垃圾筒容量提示
不知道垃圾筒攒了多少、占了多大空间。

**方案**: 标题旁显示统计："垃圾筒 · 20 个文件 · 156 MB"

## 安全

### "清理"和"恢复"视觉区分不够
`trash-action`（白色半透明）和 `trash-action-danger`（红色半透明）在 Deep Space 暗色主题下区分度一般。

**方案**: 恢复用绿色基调（`#33AABB88` 背景），清理保持红色，建立色彩直觉。
