## 1. 需求总览

### 1.1 背景

用户导入数百甚至数千张天文照片后，想知道：
- 「一年拍了多少张？哪个月最勤快？」
- 「拍得最多的是什么目标？」
- 「最长用的焦距是哪个焦段？」
- 「这一年累计曝光了多少小时？」

目前 StartTooler 没有任何统计分析功能。用户的照片数据存在 SQLite 中，具备极好的分析基础，只是缺一层可视化。

### 1.2 核心价值

| 维度 | 现在 | 将来 |
|---|---|---|
| 数据感知 | 看到照片但看不到"画像" | 仪表盘一眼看清拍摄习惯和偏好 |
| 成就感 | 数量无感知 | 累计曝光时长、年度张数——量化热爱 |
| 决策辅助 | 买新器材纯凭感觉 | 焦距分布告诉你最常用什么焦段 |
| 分享 | 只能分享单张照片 | 「年度摄影总结」有社交传播力 |

## 2. 功能需求

### 2.1 仪表盘入口

- 左侧 NavRail 新增「统计」Tab（或放在 Gallery 的子 Tab 中）
- 点击后全屏展示仪表盘（不区分左栏/图墙，独立全幅布局）

### 2.2 统计卡片（第一行：概览）

4–5 个 KPI 数字卡片，一行排列：

| 卡片 | 数据来源 | 格式 |
|---|---|---|
| 总照片数 | `COUNT(*) FROM media_files` | `14,256 张` |
| 累计曝光时长 | SUM(EXIF 曝光时间) | `386 小时 42 分钟` |
| 拍摄天数 | COUNT(DISTINCT 拍摄日期) | `124 天` |
| 拍摄目标数 | COUNT(DISTINCT AI 标签) | `15 个天体` |
| 总数据容量 | SUM(文件大小) | `1.2 TB` |

### 2.3 拍摄热力图（第二行：时间维度）

日历热力图（类似 GitHub 贡献图），展示过去 365 天每天的拍摄张数：

- X 轴 = 月份，Y 轴 = 星期
- 每个格子代表一天，颜色深浅 = 该日拍摄张数（0 = 灰色，max = 深蓝）
- Hover 某格显示日期 + 张数 + 当日主要目标
- 点击某格跳转到 Gallery 该日的时间轴

**下方附月度柱状图**：
- 按月份展示拍摄张数（如「2025-07: ██████████ 1,024 张」）
- 可切换年份
- 柱子上标注数量

### 2.4 目标排行榜（第三行：星体维度）

按拍摄数量排序的目标 TOP 15 条形图：

```
银河      ████████████████████████████████  1,890
猎户座    ██████████████████                845
月亮      ████████████                      620
星云      ██████████                        480
仙女座    ███████                           310
土星      █████                             220
...
```

- 每个目标条目可点击 → 跳转到 Gallery 该标签过滤
- 显示百分比（占总数的比例）
- 未拍过的目标灰化显示在底部

### 2.5 曝光参数分布（第四行：技术维度）

#### 2.5.1 焦距分布

按焦距段分组（从 EXIF `FocalLength` 读取）：

```
超广角  (< 24mm)    ████████████  35%
广角    (24-50mm)   ██████████    28%
标准    (50-100mm)  ██████        18%
长焦    (100-400mm) ████████      22%
超长焦  (> 400mm)   ██             5%
```

#### 2.5.2 ISO 分布

按 ISO 档位分组（如 100/200/400/800/1600/3200/6400/12800+）：

```

ISO 1600  ██████████████████████  42%

```

#### 2.5.3 曝光时长分布

按曝光时间分组（1/1000s → 30s → > 30s）：

```
< 1s      ██████         15%  (行星/月亮)
1-10s     ████████       20%  (手机星空)
10-30s    ████████████   30%  (广角星野)
> 30s     ██████████████ 35%  (深空长曝光)
```

### 2.6 年度总结页（特色功能）

每年年底/用户手动触发生成一份「年度摄影总结」：

- **封面**：年度最佳照片（按 AI 质量评分最高的一张）
- **总览数字**：拍摄张数、曝光总时长、出摊天数
- **目标地图**：今年拍了哪些天体（亮点+灰点星图）
- **月度活跃度**：哪个月最勤快
- **器材 MVP**：用得最多的机身和镜头
- **分享功能**：导出为长图（PNG）→ 发朋友圈/微博/小红书

## 3. 数据来源

| 数据 | SQL/来源 | 备注 |
|---|---|---|
| 照片数量 | `COUNT(*) FROM media_files` | 已实现 |
| 曝光时长 | EXIF `ExposureTime` 字段求和 | 需解析 EXIF |
| 拍摄日期 | EXIF `DateTimeOriginal` 分组 | 已有 `date` 列 |
| 标签 | `tags` JSON 列匹配 `SubjectTagVocabulary` | 已有 |
| 焦距 | EXIF `FocalLength` | 需解析 EXIF |
| ISO | EXIF `ISOSpeedRatings` | 需解析 EXIF |
| 文件大小 | `file_size` 列 | 已有 |
| AI 评分 | `ai_score` 列 | 已有 |

**注意**：焦距/ISO/曝光时长目前未在 SQLite 表中建列，需从 EXIF 解析。为统计性能考虑，建议在 `media_files` 表新增 3 个冗余列：

```sql
ALTER TABLE media_files ADD COLUMN focal_length_35mm REAL;  -- 35mm等效焦距
ALTER TABLE media_files ADD COLUMN iso INTEGER;
ALTER TABLE media_files ADD COLUMN exposure_time REAL;       -- 秒
```

导入时从 EXIF 提取填入，缺失则留 NULL。

### 4.2 图表实现方案

**推荐**：用 SkiaSharp 自绘（复用项目已有依赖），包体积不增加。

备选：引入 LiveChartsCore（开源 .NET 图表库，但增加依赖和包体积）。

| 图表类型 | SkiaSharp 实现思路 |
|---|---|
| KPI 卡片 | `SKCanvas.DrawRoundRect` + `DrawText` |
| 热力图 | 双重循环绘制矩形格子，颜色按 `ColorMatrix` 映射 |
| 条形图/柱状图 | `DrawRect` 按比例拉伸，顶部注文字 |
| 饼图（如需要） | `DrawArc` 按百分比扫角 |

## 5. 边界与空态

| 状态 | 表现 |
|---|---|
| 无任何照片 | 仪表盘全空态：「导入照片后将生成统计数据」 |
| 照片无 EXIF（焦距/ISO缺失） | 相关图表标注「N 张照片参数未知」 |
| 照片无 AI 标签 | 目标排行榜空态：「AI 打标后将展示目标排行」 |
| 照片量极少（< 10张） | 图表正常渲染但标注「数据量较少」 |
| 切换年份后无数据 | 图表空态显示「该年份无拍摄记录」 |

## 6. 性能考虑

| 场景 | 方案 |
|---|---|
| 万张以上照片时统计查询慢 | 统计结果缓存到内存，仅在刷新/导入/打标时重新计算 |
| 热力图 365 格 + 柱状图 + 排行同时渲染 | 各图表独立 `SKBitmap` 缓存，仅在数据变化时重绘 |
| 年度总结生成长图耗时 | 异步后台生成，显示进度条 |

---

## 7. 不做清单

| 内容 | 理由 |
|---|---|
| 与云端数据对比（本地 vs OSS） | 统计聚焦于本地数据画像，云端对比可独立规划 |
| 与其他用户对比/排行 | 当前无在线账户系统 |
| 按器材分组统计（如"这支镜头拍了多少"） | 依赖 D13 设备管理，Phase 2 |
| 导出为 Excel/CSV | 仪表盘以可视化为主，文本导出优先级低 |
| 实时统计刷新 | 数据变更频率低（导入/打标后），无需实时流 |
