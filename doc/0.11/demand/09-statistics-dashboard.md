## 1. 需求总览

### 1.1 背景

用户导入数百甚至数千张天文照片后，想知道：

- 「一年拍了多少张？哪个月最勤快？」
- 「拍得最多的是什么目标？」
- 「最长用的焦距是哪个焦段？」
- 「这一年累计曝光了多少小时？」
- 🆕 **「我在什么月相下拍过照片？」**（D12 维度）
- 🆕 **「哪些天文事件极大日我出摊了？成片如何？」**（D12 维度）

目前 StartTooler 没有任何统计分析功能。用户的照片数据存在 SQLite 中，具备极好的分析基础，只是缺一层可视化。

### 1.2 核心价值

| 维度 | 现在 | 将来 |
|---|---|---|
| 数据感知 | 看到照片但看不到"画像" | 仪表盘一眼看清拍摄习惯和偏好 |
| 成就感 | 数量无感知 | 累计曝光时长、年度张数——量化热爱 |
| 决策辅助 | 买新器材纯凭感觉 | 焦段分布告诉你最常用什么焦段 |
| 分享 | 只能分享单张照片 | 「年度摄影总结」有社交传播力 |
| 🆕 **天文复盘** | 无 | **客观展示我在什么条件下拍过什么** |
| 🆕 **数据 JOIN** | 通用天文知识（去查 Stellarium） | **基于自己照片库与天文事件库的交叉呈现** |

## 2. 功能需求

### 2.1 仪表盘入口

- 左侧 NavRail 新增「统计」Tab（或放在 Gallery 的子 Tab 中）
- 点击后全屏展示仪表盘（不区分左栏/图墙，独立全幅布局）

### 2.2 统计卡片（第一行：概览）

4–5 个 KPI 数字卡片，一行排列：

| 卡片 | 数据来源 | 格式 |
|---|---|---|
| 总照片数 | `COUNT(*) FROM media_files` | `14,256 张` |
| 累计曝光时长 | SUM(EXIF 曝光时间) | `386 小时 42 分钟` |
| 拍摄天数 | COUNT(DISTINCT 拍摄日期) | `124 天` |
| 拍摄目标数 | COUNT(DISTINCT AI 标签) | `15 个天体` |
| 总数据容量 | SUM(文件大小) | `1.2 TB` |

### 2.3 拍摄热力图（第二行：时间维度）

日历热力图（类似 GitHub 贡献图），展示过去 365 天每天的拍摄张数：

- X 轴 = 月份，Y 轴 = 星期
- 每个格子代表一天，颜色深浅 = 该日拍摄张数（0 = 灰色，max = 深蓝）
- Hover 某格显示日期 + 张数 + 当日主要目标
- 点击某格跳转到 Gallery 该日的时间轴

**下方附月度柱状图**：

- 按月份展示拍摄张数（如「2025-07: ██████████ 1,024 张」）
- 可切换年份
- 柱子上标注数量

### 2.6 年度总结页（特色功能）

每年年底/用户手动触发生成一份「年度摄影总结」：

- **封面**：年度质量评分最高的照片
- **总览数字**：拍摄张数、曝光总时长、出摊天数
- **目标地图**：今年拍了哪些天体（亮点+灰点星图）
- **月度活跃度**：哪个月最勤快
- **器材 MVP**：用得最多的机身和镜头
- **分享功能**：导出为长图（PNG）→ 发朋友圈/微博/小红书

### 2.7 🌙 天文复盘维度（原 D12 合并至此）🆕

> **定位**：纯描述型。不预测、不提醒、不联网、不评价、不建议。
> 回答的唯一问题：**「你拍过什么」**。

#### 2.7.1 三大数据源

| 数据源 | 用途 | 是否本地 |
|---|---|---|
| `sessions` 表（D08） | 出摊日期、时长、照片数、平均质量 | ✅ |
| `media_files` 表（EXIF） | 拍摄时间、GPS、参数 | ✅ |
| Meeus 算法 | 历史月相回溯 | ✅ |
| 嵌入事件 TOML | 历史事件模板（流星雨/日月食） | ✅ |

#### 2.7.2 月相偏好画像 🆕

```
🌑 新月  ：出摊 8 次 · 成片 42 张/次 · 质量 4.3★
🌒 蛾眉月：出摊 5 次 · 成片 28 张/次 · 质量 3.8★
🌓 上弦  ：出摊 4 次 · 成片 18 张/次 · 质量 3.2★
🌕 满月  ：出摊 7 次 · 成片 8 张/次  · 质量 2.1★
🌗 下弦  ：出摊 3 次 · 成片 12 张/次 · 质量 2.8★
🌘 残月  ：出摊 2 次 · 成片 25 张/次 · 质量 3.6★
```

**SQL 聚合**：

```sql
SELECT 
    CASE 
        WHEN moon_age < 1.0 OR moon_age > 28.5 THEN '新月'
        WHEN moon_age < 6.0 THEN '蛾眉月'
        WHEN moon_age < 9.0 THEN '上弦'
        WHEN moon_age < 13.0 THEN '盈凸'
        WHEN moon_age < 16.5 THEN '满月'
        WHEN moon_age < 20.0 THEN '亏凸'
        WHEN moon_age < 23.0 THEN '下弦'
        ELSE '残月'
    END AS phase_category,
    COUNT(*) AS session_count,
    SUM(photo_count) AS total_photos,
    AVG(avg_quality_score) AS avg_quality,
    SUM(CASE WHEN avg_quality_score >= 4.0 THEN 1 ELSE 0 END) * 1.0 
        / COUNT(*) AS high_ratio
FROM sessions
WHERE project_id = ? AND moon_age IS NOT NULL
GROUP BY phase_category
ORDER BY avg_quality DESC;
```

## 3. 数据来源

### 3.1 基础统计数据

| 数据 | SQL/来源 | 备注 |
|---|---|---|
| 照片数量 | `COUNT(*) FROM media_files` | 已实现 |
| 曝光时长 | EXIF `ExposureTime` 字段求和 | 需解析 EXIF |
| 拍摄日期 | EXIF `DateTimeOriginal` 分组 | 已有 `date` 列 |
| 标签 | `tags` JSON 列匹配 `SubjectTagVocabulary` | 已有 |
| 焦距 | EXIF `FocalLength` | 需解析 EXIF |
| ISO | EXIF `ISOSpeedRatings` | 需解析 EXIF |
| 文件大小 | `file_size` 列 | 已有 |
| AI 评分 | `ai_score` 列 | 已有 |

**注意**：焦距/ISO/曝光时长目前未在 SQLite 表中建列，需从 EXIF 解析。为统计性能考虑，建议在 `media_files` 表新增 3 个冗余列：

```sql
ALTER TABLE media_files ADD COLUMN focal_length_35mm REAL;  -- 35mm等效焦距
ALTER TABLE media_files ADD COLUMN iso INTEGER;
ALTER TABLE media_files ADD COLUMN exposure_time REAL;       -- 秒
```

导入时从 EXIF 提取填入，缺失则留 NULL。

#### 3.2.3 Meeus 算法（仅月相）

**仅实现一个函数**：`ComputeMoonAge(DateTime utc) → double`

- 精度：±5 分钟（桌面应用足够）
- 实现：自写简化版（< 100 行）或使用 NuGet 包（如 [MeeusSharp](https://github.com/ingves/MeeusSharp)）
- **不计算**：太阳位置、行星位置、月出月落（v0.2 范围外）

> 注：v0.2 简化了原 D12 的算法范围。如未来需要月出月落展示，再补充实现。

### 4.2 图表实现方案

**推荐**：用 SkiaSharp 自绘（复用项目已有依赖），包体积不增加。

备选：引入 LiveChartsCore（开源 .NET 图表库，但增加依赖和包体积）。

| 图表类型 | SkiaSharp 实现思路 |
|---|---|
| KPI 卡片 | `SKCanvas.DrawRoundRect` + `DrawText` |
| 热力图 | 双重循环绘制矩形格子，颜色按 `ColorMatrix` 映射 |
| 条形图/柱状图 | `DrawRect` 按比例拉伸，顶部注文字 |
| 饼图（如需要） | `DrawArc` 按百分比扫角 |
| 🆕 月相条形图 | 条形图变体，左侧 emoji 图标 |

## 5. 边界与空态

| 状态 | 表现 |
|---|---|
| 无任何照片 | 仪表盘全空态：「导入照片后将生成统计数据」 |
| 照片无 EXIF（焦距/ISO缺失） | 相关图表标注「N 张照片参数未知」 |
| 照片无 AI 标签 | 目标排行榜空态：「AI 打标后将展示目标排行」 |
| 照片量极少（< 10张） | 图表正常渲染但标注「数据量较少」 |
| 切换年份后无数据 | 图表空态显示「该年份无拍摄记录」 |
| 🆕 会话无月相字段 | 仪表盘天文区域标注「月相数据待补全」+ [立即计算] 按钮 |
| 🆕 无任何天文事件匹配 | 「事件回顾」模块空态：「暂无天文事件匹配记录」 |
| 🆕 出摊 < 5 次 | 月相偏好图标注「数据量较少，X 次出摊的样本不足以显示规律」 |

## 6. 性能考虑

| 场景 | 方案 |
|---|---|
| 万张以上照片时统计查询慢 | 统计结果缓存到内存，仅在刷新/导入/打标时重新计算 |
| 热力图 365 格 + 柱状图 + 排行同时渲染 | 各图表独立 `SKBitmap` 缓存，仅在数据变化时重绘 |
| 年度总结生成长图耗时 | 异步后台生成，显示进度条 |
| 🆕 月相回溯（启动时全量补全） | 仅在 `moon_age IS NULL` 时计算，结果写回 sessions 表 |
| 🆕 嵌入事件库加载 | < 10KB，启动时一次性加载到内存 |

## 8. 不做清单

### 8.1 基础统计

| 内容 | 理由 |
|---|---|
| 与云端数据对比（本地 vs OSS） | 统计聚焦于本地数据画像，云端对比可独立规划 |
| 与其他用户对比/排行 | 当前无在线账户系统 |
| 按器材分组统计（如"这支镜头拍了多少"） | 依赖 D13 设备管理，Phase 2 |
| 导出为 Excel/CSV | 仪表盘以可视化为主，文本导出优先级低 |
| 实时统计刷新 | 数据变更频率低（导入/打标后），无需实时流 |

### 8.2 天文复盘 🆕（v0.2 精简）

**永久不做**：

| 内容 | 理由 |
|---|---|
| 月相预测 / 未来月相日历 | 纯描述型，不需要 |
| 未来事件列表 | 纯描述型，不需要 |
| 未来 3 年事件库维护 | 不需要预测 |
| 拍摄条件评分（预测当晚） | 纯描述型 |
| 关注目标倒计时 | 纯描述型 |
| 拍摄建议 / "最适合拍什么" | **v0.3 新增：纯描述型定位，不做主观建议** |
| "错过的窗口" 等评价性表达 | **v0.3 新增：避免使用"错过"等带价值判断的措辞** |
| 与其他用户对比 / 排行 | 客观数据不存在可比基准 |
| 实时星图（Stellarium 类） | 竞争激烈，重复造轮 |
| 天气集成 | 留给 v0.14 |
| 极光预报 | 用户群体不匹配 |

| 彗星新发现 | 嵌入数据足够 |
| 彗星轨道在线获取 | 不做 |
| 系统通知 / 提醒 | 纯描述型，不打扰用户 |
| **任何外部 API** | 全部不做，纯本地 |
| 行星位置计算 | v0.3 范围内不实现 |
