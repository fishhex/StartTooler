## 1. 需求总览

### 1.1 背景

天文摄影后期处理中，对比是核心操作场景：

- **选片**：「这 3 张银河，哪张对焦最准？」
- **处理前后**：「叠加/增强之后到底提升多少？」
- **参数对比**：「ISO 800 vs ISO 1600，噪点差别有多大？」
- **版本对比**：「第一版和第二版后期，哪个星点更细？」

当前只能通过系统默认图片查看器打开两张照片，手动切换窗口对比，效率极低。灯箱每次只能看一张。

### 1.2 核心价值

对比工具是 v0.11「一键合成」和 v0.10 AI 增强的**下游闭环**——合成/增强之后，用户需要对比原图和结果来评估效果。没有对比，合成的价值感知大打折扣。

## 2. 功能需求

### 2.1 三种对比模式

#### 模式 1：并排对比（Side by Side）

两张照片左右水平排列。

```
┌──────────────────────────────────────────┐
│  📷 M42_原图.NEF           │  📷 M42_叠加.PNG           │
│ ┌────────────────────────┐ │ ┌────────────────────────┐ │
│ │                        │ │ │                        │ │
│ │      原图（左）         │ │ │    叠加后（右）         │ │
│ │                        │ │ │                        │ │
│ └────────────────────────┘ │ └────────────────────────┘ │
│  ISO 800  120s  300mm     │  ISO 800  120s  300mm     │
├──────────────────────────────────────────────────────────┤
│  [🔀 左右交换]  [↔️ 同步缩放]  [🔲 匹配大小]            │
└──────────────────────────────────────────────────────────┘
```

**特性**：
- 同步缩放：鼠标滚轮缩放时两张图同步放大到相同比例
- 同步拖动：拖动一张图平移时另一张同步移动（匹配相同区域）
- 左右交换：一键交换左右位置
- 匹配大小：根据分辨率差异自动调整显示大小使两张图宽度一致

#### 模式 2：滑动对比（Split View / Slider）

两张图上下重叠，拖动滑块展示 A/B 分界。

```
┌──────────────────────────────────────────┐
│  ← 拖动滑块 →                             │
│ ┌────────────────────────────────────────┐│
│ │              │                         ││
│ │   增强前     │    增强后               ││
│ │  (左半)     │   (右半)                ││
│ │              │                         ││
│ └────────────────────────────────────────┘│
│        ▲                                   │
│        └── 可拖动的分界线                   │
│                                            │
│  [⬅️➡️ 拖动滑块对比增强效果]              │
└──────────────────────────────────────────┘
```

**特性**：
- 滑块初始位置居中，可拖动到任意位置
- 两张图同步缩放
- 上方显示标签：左侧 = 图 A 文件名，右侧 = 图 B 文件名

#### 模式 3：闪烁对比（Blink）

两张图在同一位置交替快速显示，以察觉细微差异（如星点位移、噪点水平）。

```
┌──────────────────────────────────────────┐
│  🔄 闪烁中（200ms/帧）                    │
│ ┌────────────────────────────────────────┐│
│ │                                        ││
│ │      A ↔ B 交替显示                    ││
│ │                                        ││
│ └────────────────────────────────────────┘│
│                                            │
│  闪烁速度: [慢 ▬▬▬●▬▬▬ 快] 200ms         │
│  [⏸️ 暂停]  [A] [B] [A+B 并排]            │
└──────────────────────────────────────────┘
```

**特性**：
- 可调闪烁速度（100ms–1000ms 范围，默认 200ms）
- 暂停在 A 或 B 图
- 暂停后可一键切到「并排对比」模式查看细节
- 主要用于天文照片中对齐与否、噪点差异的肉眼判断

### 2.2 入口与触发

| 入口 | 触发方式 |
|---|---|
| Gallery 多选 | 选中 2 张照片 → 右键「对比」或工具栏按钮 |
| 灯箱预览 | 灯箱内「对比」按钮 → 弹出文件选择器再选一张 |
| 合成/增强完成后 | 结果展示页自动出现「与原图对比」按钮 → 直接进入滑动对比模式 |
| 增强历史面板 | 点击历史版本 →「与当前版本对比」 |

### 2.3 比较信息展示

每张图下方或侧边显示可折叠的信息条：

| 字段 | 来源 |
|---|---|
| 文件名 | `file_name` |
| 拍摄时间 | EXIF `DateTimeOriginal` |
| 曝光参数 | ISO / 光圈 / 快门 / 焦距（EXIF） |
| 文件大小/分辨率 | `file_size` / 图片像素尺寸 |
| AI 标签 | `tags` JSON |
| AI 评分 | `ai_score` |
| 叠加/增强来源 | 派生关系标注 |

如果两张图的参数存在差异（如 ISO 不同），差异项用**橙黄色高亮**。

### 2.4 快捷操作

| 操作 | 快捷键 | 行为 |
|---|---|---|
| 切换对比模式 | `1` / `2` / `3` | 并排 / 滑动 / 闪烁 |
| 缩放 | `Ctrl + 滚轮` | 两张图同步缩放 |
| 重置缩放 | `Ctrl + 0` | 恢复 100% |
| 适配窗口 | `Ctrl + 9` | 缩放至刚好充满窗口 |
| 关闭对比 | `Esc` | 返回 Gallery 或灯箱 |
| 暂停闪烁 | `Space` | 暂停在当前帧 |

## 3. 数据需求

无需新增数据库表。对比工具完全是视图层操作，所需数据全部来自已有 `media_files` 表。

需要新增的 ViewModel 方法：

```csharp
// CompareViewModel（新增）
public partial class CompareViewModel : ObservableObject
{
    [ObservableProperty] private MediaFile _leftFile;
    [ObservableProperty] private MediaFile _rightFile;
    [ObservableProperty] private CompareMode _mode;    // SideBySide / Split / Blink
    [ObservableProperty] private double _zoomLevel = 1.0;
    [ObservableProperty] private double _splitPosition = 0.5;  // 滑动对比滑块位置
    [ObservableProperty] private int _blinkIntervalMs = 200;   // 闪烁间隔
    [ObservableProperty] private bool _syncPan = true;         // 同步平移
}
```

### 图像加载与缩放

- 两图需要**独立解码**并缓存到 `SKBitmap`（复用 SkiaSharp）
- 缩放在两张图上同步应用相同的 `scale` 矩阵
- 并排对比模式下，两张图独立平移到各自的中心对齐区域

## 4. UI 布局

### 4.1 对比窗口

建议复用灯箱窗口的方式（独立 `Window`），最大化利用屏幕空间：

```
┌──────────────────────────────────────────────────────────┐
│  照片对比                                   [_][□][×]   │
├──────────────────────────────────────────────────────────┤
│  模式: [并排] [滑动] [闪烁]  速度: [━━●━━━]   [交换]   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│                    照片 A  │  照片 B                      │
│                          │                               │
│                          │                               │
│                    (同步缩放/拖动)                        │
│                          │                               │
│                          │                               │
├──────────────────────────────────────────────────────────┤
│ 左: DSC_0001.NEF  ISO800 f/2.8 120s  │ 右: DSC_0002.NEF  ISO1600 f/2.8 120s  │
└──────────────────────────────────────────────────────────┘
```

### 4.2 信息差异高亮

当左右图参数存在差异时，差异字段用不同背景色标注：

```
左: DSC_0001.NEF  [ISO 800] f/2.8 120s
右: DSC_0002.NEF  [ISO 1600] f/2.8 120s     ← ISO 差异标黄
```

## 5. 边界与异常

| 场景 | 处理 |
|---|---|
| 两张图分辨率差异巨大（如 1000px vs 8000px） | 并排模式以较小图的显示区域为准，大图等比缩小适配 |
| 两张图形状差异大（横幅 vs 竖幅） | 并排模式各自保持宽高比，居中显示 |
| 其中一张图文件损坏/无法加载 | 该侧显示错误占位：「无法加载 XXX.NEF」，另一侧正常显示 |
| 未选中 2 张照片时触发对比 | 如从灯箱入口，弹出选择第二张的对话框 |
| 选中超过 2 张 | 只取前 2 张，toast 提示「对比仅支持 2 张，已取前两张」 |
| 闪烁对比中切换到其他窗口 | 自动暂停闪烁，回到窗口后恢复 |

## 6. 与已有功能的关系

| 功能 | 关系 |
|---|---|
| Lightbox 灯箱 | 对比窗口可视为「双图灯箱」，复用灯箱的缩放/拖动逻辑 |
| 一键合成 | 合成完成后「与原图对比」按钮直接进入滑动对比 |
| 增强历史 | 历史版本之间可两两对比 |
| Gallery 多选 | 多选 2 张后右键菜单出现「对比」选项 |

---

## 7. 不做清单

| 内容 | 理由 |
|---|---|
| 3 张以上对比 | 核心场景是 A/B 对比，多图对比增加 UI 复杂度且需求少 |
| 像素级差异叠加（Difference Blend） | 实现复杂，一般用户用闪烁对比即可感知差异 |
| 对比结果的保存/导出 | 对比是临时浏览工具，不产生新文件 |
| 视频对比 | 视频对比需要帧对齐播放，复杂度极高 |
