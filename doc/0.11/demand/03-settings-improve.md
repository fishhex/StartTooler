# Settings（设置）UI & 交互改进

> 对应 `Views/SettingsView.axaml`，当前 421 行，3 个 Tab（通用 / OSS配置 / AI）。

## 通用 Tab

### 项目目录选择器
Flyout 内"最近使用"列表是硬编码占位代码（`IsVisible="False"`），实际不显示。

**方案**: 从 `config.db` 的目录历史中读取真实最近目录列表，动态生成 `MenuItem`。

### 无表单验证
所有输入字段无即时或提交时的校验反馈。

**方案**:
- `TextBox` 失焦（`LostFocus`）时校验格式
- 非法值：边框变红 + 下方显示错误提示文字
- OSS 密钥为空时保存按钮置灰
- AI API Key 格式校验（常见前缀如 `sk-` 检查）

### 保存无成功反馈
点击保存后，`IsDirty` 变 false 但无任何视觉或文字反馈。

**方案**: 保存成功后：
- 保存按钮短暂显示 "已保存 ✓"（1.5 秒后恢复）
- 或底部弹出绿色 toast "设置已保存"

### 无设置导入/导出
重装或换机器需要重新配置。

**方案**: 通用 Tab 底部加两个按钮：
- "导出配置" — 将 `config.db` 导出为 JSON 文件（不含密钥）
- "导入配置" — 从 JSON 恢复设置，密钥字段需手动重填

## OSS Tab

### 供应商 ComboBox 不可交互
`IsHitTestVisible="False"`，虽然目前只有阿里云，但看起来像是 bug 而非设计意图。

**方案**: 如短期内无其他供应商，改为只读 `TextBlock` 显示"阿里云 OSS"，不展示下拉框。

### 无连接测试
AI Tab 有"测试连接"，OSS 没有。

**方案**: 加"测试连接"按钮，调用 `ListObjects` 或 `HeadBucket` 验证配置正确性，反馈成功/失败。

### AccessKeySecret 明文风险
两个密钥并排显示，Secret 有 `PasswordChar` 但值仍在 ViewModel 中以字符串明文传输。

**方案**: 短期：Secret 字段默认 `PasswordChar="•"`，加眼睛切换按钮（同 AI Key）。长期：考虑 `SecureString`。

### 路径前缀无自动补全
用户可能不知道路径格式。

**方案**: `Watermark` 改为更具体的示例（如 `astrophotos/`），失焦时自动补末尾 `/`。

## AI Tab

### 厂商选择影响范围不清
切换厂商时自动填入 Base URL 和模型推荐列表，用户手填的 API Key 不应被覆盖但行为不明确。

**方案**: 切换厂商时弹出提示："切换厂商将重置 Base URL 和模型推荐列表，API Key 保持不变"，加"不再提示"复选框。

### 测试连接不可自定义
测试 prompt 固定为代码中写死的内容。

**方案**: 加可编辑的测试 prompt 字段（`TextBox`，`AcceptsReturn="True"`，默认值 "请分析这张天文照片的主体、质量和拍摄参数"），允许用户自定义测试内容以验证模型答复质量。

### 模型下拉框交互不直观
`IsEditable="True"` 的 ComboBox，可下拉选也可手输，但下拉项和手输值之间的边界不清晰。

**方案**: 加说明文字："可从下拉列表选择推荐模型，也可直接输入任意模型名"

### 无用量统计
不知道 AI 调用了多少次、花了多少钱。

**方案**: AI Tab 底部显示统计（需在 AITagger 中埋点）：
- 本月调用次数
- 本月 token 用量（如有返回）
- 上次调用时间

## 全局

### 设置导航守卫仅确认"放弃更改"
`IsDirty` 时离开页面弹确认框，但没有"保存并离开"选项。

**方案**: 三选对话框："保存并离开" / "放弃更改" / "取消"
