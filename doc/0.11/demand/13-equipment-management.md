## 1. 需求总览

### 1.1 背景

天文摄影涉及大量器材：望远镜（折射/反射/折反）、赤道仪、相机（冷冻/单反/微单）、滤镜（LRGB/Ha/SII/OIII）、导星系统、平场镜、减焦镜……用户通常拥有多套组合，但从照片本身很难完整回溯拍摄时用了什么。

当前 StartTooler 完全不管器材信息。EXIF 中能读到的只有相机型号、镜头焦距和光圈，无法记录望远镜、赤道仪和滤镜。

### 1.2 核心价值

| 维度 | 现在 | 将来 |
|---|---|---|
| 拍摄复盘 | 不记得这张 M42 用的是什么望远镜 | 每张照片可关联器材组合 |
| 器材利用率 | 不知道哪套组合用得最多 | 统计数据告诉你器材 MVP |
| 购买决策 | 凭感觉决定买什么滤镜 | 数据告诉你最常用的焦段和缺失的焦距 |
| 新手参考 | 无 | 拍摄会话报告含完整器材列表，可分享给朋友参考 |

## 2. 功能需求

### 2.1 器材分类

支持以下器材类型：

| 类型 | 示例 | 关键属性 |
|---|---|---|
| 望远镜/镜头 | RedCat 51, C8HD, Sigma 14mm f/1.8 | 品牌、型号、口径(mm)、焦距(mm)、焦比 |
| 相机 | ZWO ASI2600MC Pro, Sony A7R5, Canon EOS R5 | 品牌、型号、传感器类型(CMOS/CCD)、像素、是否冷冻 |
| 赤道仪 | ZWO AM5, iOptron CEM70, 信达 HEQ5 Pro | 品牌、型号、载重(kg)、类型(谐波/德式) |
| 滤镜 | Antlia 3nm Ha, Optolong L-eXtreme, UV/IR Cut | 品牌、型号、类型(LRGB/Ha/SII/OIII/SHO)、尺寸(1.25"/2"/36mm) |
| 导星系统 | ZWO ASI120MM Mini, ZWO OAG | 品牌、型号、导星镜焦距（如独立导星镜） |
| 其他配件 | 平场镜、减焦镜、巴罗镜、电源、除雾带等 | 品牌、型号、备注 |

### 2.3 添加/编辑器材

每个器材类型的编辑表单包含：

**望远镜/镜头**：

| 字段 | 类型 | 必填 |
|---|---|---|
| 品牌 | 文本 | 是 |
| 型号 | 文本 | 是 |
| 口径 (mm) | 数字 | 否 |
| 焦距 (mm) | 数字 | 否 |
| 焦比 | 自动计算 = 焦距/口径 | — |
| 类型 | 下拉（折射/反射/折反/相机镜头） | 否 |
| 备注 | 文本 | 否 |

**相机**：

| 字段 | 类型 | 必填 |
|---|---|---|
| 品牌 | 文本 | 是 |
| 型号 | 文本 | 是 |
| 传感器类型 | 下拉（CMOS/CCD） | 否 |
| 传感器尺寸 | 下拉（全画幅/APS-C/M4/3/1"/其他） | 否 |
| 像素 | 文本（如 `6248×4176`） | 否 |
| 像元大小 | 数字 + 单位 (μm) | 否 |
| 是否冷冻相机 | 开关 | 否 |
| 备注 | 文本 | 否 |

**赤道仪**、**滤镜** 等类似，各有专属字段。

### 2.4 器材组合（Rig）

用户可将多个器材组合成一个「常用组合」，方便在拍摄会话中快速选择。

```
Rig: "轻便星野套装"
├── 望远镜: RedCat 51
├── 相机: ASI2600MC Pro
├── 赤道仪: ZWO AM5
└── 滤镜: UV/IR Cut
```

- 每个组合可命名、编辑、删除
- 删组合不删器材
- 删器材时检测是否被组合引用，有引用则提示

### 2.5 照片与器材关联

#### 2.5.1 手动关联（Phase 1）

- 拍摄会话详情中选择器材组合（见 D08 §2.3.3）
- 该会话内所有照片自动标记为此组合

#### 2.5.2 自动匹配（Phase 2）

- 导入照片时从 EXIF `Model` 自动匹配相机库中的型号
- 从 EXIF `FocalLength` 匹配焦距相近的望远镜/镜头
- 匹配结果作为建议，用户确认后关联

## 3. 数据需求

### 3.1 器材表 `equipment`

```sql
CREATE TABLE equipment (
    id         TEXT PRIMARY KEY,          -- GUID
    type       TEXT NOT NULL,             -- telescope / camera / mount / filter / guider / accessory
    brand      TEXT NOT NULL,
    model      TEXT NOT NULL,
    specs      TEXT NOT NULL DEFAULT '{}', -- JSON，各类型的专属字段
    notes      TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

`specs` JSON 按类型不同：

```json
// 望远镜
{"aperture_mm": 51, "focal_length_mm": 250, "focal_ratio": 4.9, "type": "refractor"}
// 相机
{"sensor_type": "CMOS", "sensor_size": "APS-C", "pixels": "6248×4176", "pixel_size_um": 3.76, "cooled": true}
// 赤道仪
{"payload_kg": 13, "type": "harmonic"}
// 滤镜
{"filter_type": "Ha", "bandwidth_nm": 3, "size": "2\""}
```

### 3.2 组合表 `rigs`

```sql
CREATE TABLE rigs (
    id         TEXT PRIMARY KEY,
    name       TEXT NOT NULL,
    telescope_id TEXT,
    camera_id  TEXT,
    mount_id   TEXT,
    filter_ids TEXT NOT NULL DEFAULT '[]',  -- JSON 数组，支持多滤镜
    notes      TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.3 根据 EXIF 自动匹配（辅助查询）

```csharp
// IEquipmentRepository
Task<Equipment?> FindCameraByModelAsync(string exifModel, CancellationToken ct);
Task<Equipment?> FindTelescopeByFocalLengthAsync(double focalLengthMm, CancellationToken ct);
```

## 4. UI 交互

### 4.1 器材列表

- 按类型分组折叠
- 每项显示：品牌 + 型号 + 关键参数缩略
- 右键操作：编辑 / 删除 / 查看关联照片数
- 删除时提示「该 XX 被 N 个组合引用，删除后组合将移除该项」

### 4.2 组合管理

- 组合列表在器材 Tab 中展示（或在器材列表下方）
- 创建组合：「新建组合」→ 逐一选择望远镜/相机/赤道仪/滤镜
- 各字段非必填：不勾选赤道仪意味着这组可以是「星野赤道仪未记录的夜间拍摄」或「星野摄影（仅三脚架）」
- 组合在拍摄会话中选择后，该会话内所有照片继承此器材信息

## 5. 边界与空态

| 状态 | 表现 |
|---|---|
| 无任何器材 | 器材 Tab 空态：「添加你的望远镜、相机等器材」 |
| 器材被删除但被组合引用 | 提示「该器材被 N 个组合引用，删除后组合将移除该项」 |
| 导入的照片相机型号未录入 | 照片不关联相机，状态栏不显示器材 |
| 同一型号多台（如两台 ASI2600MC） | 支持添加多条同名器材，通过序号或备注区分 |

## 7. 不做清单

| 内容 | 理由 |
|---|---|
| 设备连接/控制（ASCOM/INDI） | 桌面应用不做设备驱动层，那是 NINA/SharpCap 的领域 |
| 快门寿命实时统计 | 需要用户在应用内计数，EXIF 无法可靠追踪 |
| 二手器材交易/行情 | 非核心功能 |
| 器材校准数据（暗场库、平场库） | 暗场管理是独立大需求 |
| 社区共享器材配置 | 无账户系统 |
