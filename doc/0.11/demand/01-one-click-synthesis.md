## 1. 需求总览

StartTooler 目前解决的是**管和看**（归档、预览、备份、AI 打标）。用户拍完素材回到软件后，还有**出片**的最后一步——把素材变成一张拿得出手的成片。

本需求覆盖两大场景：

| 场景 | 输入 | 输出 | 目标用户 |
|---|---|---|---|
| **场景一：月面/行星视频叠加** | .SER / .AVI / .MP4 视频（数千帧） | 单张高锐度静态图 | 行星相机 + 望远镜用户 |
| **场景二：夜空照片一键增强** | 单张手机长曝光星空照片（JPEG/RAW） | 增强后成片 | 普通手机拍星用户 |

两个场景的核心设计理念一致：**少操作、快出片、强默认**——默认参数就能得到不错的结果，有需求的用户可以展开高级选项微调。

## 2. 场景一：月面/行星视频叠加

### 2.1 用户故事

> 我用行星相机对着木星拍了一段 3 分钟的视频（几千帧），导入 StartTooler 后归档在了今天的日期下。现在我想把这段视频叠加成一张清晰的照片，不想再拷出去用 AutoStakkert + RegiStax 来回折腾。

**痛点**：现有工作流需要在多款软件之间导入导出，视频文件大、步骤多、参数复杂，新手劝退。

**期望**：在 StartTooler 时间线上找到那段视频 → 点「叠加」→ 等一会儿 → 成片出现。

### 2.2 子场景

#### 2.2A 单段视频叠加（主力场景）

一段视频 → 一张锐利成片。适合木星、土星、火星等行星，以及覆盖整个视场的月球局部特写。

**处理流程**：
1. 逐帧质量分析排序（大气抖动 → 部分帧清晰、部分帧模糊）
2. 自动选取最佳参考帧
3. 按比例挑选前 N% 高质量帧
4. 对齐所有选中帧（亚像素精度）
5. 叠加（均值 / 中位数）
6. 锐化（小波 / 反卷积）

#### 2.2B 月面多段拼接（进阶场景）

当月球大于望远镜视场时，用户拍摄多段视频覆盖月面不同区域。

**处理流程**：
1. 每段视频独立完成叠加 → 得到 N 张局部清晰图像
2. 自动检测重叠区域 → 拼接为完整月面
3. 输出全景月面成片

### 2.3 操作模式

#### 简单模式（默认）

选中视频 → 点击「开始叠加」→ 全自动完成。用户**不接触任何参数**。

自动策略：
- 参考帧：自动评估，选择综合质量最高的帧
- 保留比例：自动判断（视宁度好 → 多保留，视宁度差 → 少保留）
- 锐化强度：自动适中

#### 高级模式（用户可展开）

在叠加前可展开高级面板，手动调整：

| 参数 | 说明 | 默认值 |
|---|---|---|
| 参考帧 | 选择第几帧作为对齐基准 | 自动 |
| 保留比例 | 保留前 X% 的高质量帧 | 自动 |
| 叠加方式 | 均值 / 中位数 | 均值 |
| 锐化强度 | 0–100 滑块 | 自动 |
| 输出格式 | PNG / TIFF (16bit) / JPEG | PNG |

高级面板中展示**帧质量分布曲线**，帮助用户直观理解视宁度变化，决定保留比例的取舍。

### 2.4 UI 交互

**入口**：
- 时间线上右键视频 →「叠加合成」
- 工具栏按钮（选中视频时启用）
- 双击视频进入详情页 → 叠加按钮

**叠加中**：
- 进度条展示当前阶段（分析帧质量 → 对齐 → 叠加 → 锐化）
- 显示预估剩余时间
- 可取消

**完成后**：
- 成片出现在左侧时间线，与源视频同一天
- 标注来源关系（「叠加自：xxx.avi」）
- 源视频不受影响

### 2.5 边界与异常

| 情况 | 处理 |
|---|---|
| 视频 < 100 帧 | 提示「素材帧数不足（N 帧），建议至少拍摄 500 帧以上」，允许继续但降低期望 |
| 视频质量极差（全程大气抖动严重） | 完成叠加后附提示「视宁度较差，建议在更稳定的大气条件下重新拍摄」 |
| 输入格式不支持 | 提示「不支持的视频格式：.xxx」，列出支持格式 |
| 拼接时重叠区域不足 | 提示「第 X 段与第 Y 段重叠区域过小，无法自动拼接」，建议检查拍摄覆盖 |
| 多段拼接中某段叠加失败 | 跳过该段，拼接其余部分，标注缺失区域 |
| 用户中途取消 | 保留已完成的分析结果缓存，下次可恢复 |

## 3. 场景二：夜空照片一键增强

### 3.1 用户故事

> 露营时用手机架在小三脚架上，长曝光 15 秒拍了张银河。回到家用 StartTooler 归档了，但照片灰蒙蒙的、噪点多、银河不明显。我不懂 PS 也不会调参数，就想一键让星空照片变好看。

**痛点**：手机长曝光星空照普遍存在噪点多、颜色偏、银河不突出等问题，普通用户的修图门槛高。

**期望**：打开照片 → 点「一键增强」→ 前后对比 → 满意保存。

### 3.2 处理流程

#### 自动分析与处理（不可见）

```
照片输入
  ↓
[1] 场景识别 ──→ 是星空照？ ──否──→ 通用增强（提亮、降噪、调色温）
  │   是
  ↓
[2] 天地分离 ──→ 有地景？ ──否──→ 全图星空增强
  │   是
  ↓
[3] 光害检测 ──→ 有光害梯度？ ──是──→ 梯度去除
  │
  ↓
[4] 分区处理
  ├── 天空区域：拉升对比度、提亮星点、银河云气增强、降噪
  └── 地景区域：适度提亮、保持自然观感、边缘融合
  ↓
[5] 全局优化：色调统一、输出锐化
  ↓
呈现结果
```

#### 各步骤说明

| 步骤 | 说明 |
|---|---|
| **场景识别** | 判断照片中是否有星点特征。有星点 → 走星空增强流程；纯城市夜景或无特征 → 走通用增强流程 |
| **白平衡校正** | 消除光害造成的暖色偏色（钠灯黄光、城市橙光），还原夜空应有的冷色调 |
| **光害梯度去除** | 检测地平线方向的城市光害渐变，自动建模并去除，让天空背景均匀 |
| **天地分离** | 自动识别天空与地景分界线。天空区域拉升对比度、增强银河结构；地景区域保守处理，保持自然感。分界线做羽化过渡 |
| **降噪** | 在保留星点细节的前提下，压制 ISO 高感噪声 |
| **银河/星云增强** | 拉伸暗部细节，让银河尘埃带和弥散星云结构显现 |

### 3.3 用户操作

1. 在时间线上选中照片 → 工具栏点击「一键增强」
2. 等待处理完成（展示进度动画）
3. 结果呈现，默认展示**前后对比视图**（左右分屏或滑块对比）
4. 用户拖动**「增强强度」滑块**，在「原图」到「最大增强」之间实时调整
5. 满意后点击「保存」，生成增强副本（原图保留）

### 3.4 强度滑块

不是简单的透明度叠加，而是**语义级强度调整**：

| 强度 0% | 强度 50% | 强度 100% |
|---|---|---|
| 原图 | 适中增强（默认位置） | 最大增强 |

滑块每档控制：
- 对比度拉升程度
- 降噪力度
- 银河结构增强量
- 饱和度提升量

默认滑块停在 **50%**（中庸、自然），用户可根据偏好左右调整。

### 3.5 边界与异常

| 情况 | 处理 |
|---|---|
| 照片中无星点（纯城市夜景、室内） | 场景识别为非星空，走通用增强（提亮、降噪、调色温），不强行用星空算法 |
| 照片中有强月亮 | 自动降低天空增强强度，避免月亮区域过曝。如有月面细节，做高光保护 |
| 照片纯星野、无地景 | 跳过天地分离步骤，全图按天空策略处理 |
| 地景分界线识别失败 | 降级为全图增强，不做分区处理，同时提示「未能识别地景区域，已进行整体增强」 |
| 光害极严重（夜空近乎全橙） | 尽力去除，完成后提示「光害较重，建议前往暗夜环境拍摄以获得更好效果」 |
| RAW 文件输入 | 优先利用 RAW 的高位深信息，处理结果可保存为 16bit TIFF |
| 批量处理 | 支持多选照片后一键批量增强，每张独立分析处理 |

## 4. 跨场景共性设计

### 4.1 结果管理

- **非破坏性**：源文件（视频/照片）始终保留，处理结果为**派生副本**
- **来源追溯**：派生结果标注与源文件的关系（「叠加自」「增强自」），在时间线上可见
- **可重新处理**：源文件可多次处理生成不同参数的结果
- **不自动打标签**：增强/叠加结果不自动添加 AI 标签，用户手动管理
- **不自动上传 OSS**：增强/叠加结果不自动上传云存储，用户手动选择上传

### 4.2 进度反馈

两种场景都是计算密集型操作，需提供：
- 当前处理阶段描述
- 进度百分比或步骤指示
- 预估剩余时间
- 取消按钮（取消后保留中间产物，下次可恢复）

### 4.3 硬件感知

- 检测是否具备 GPU 加速条件，优先使用 GPU
- GPU 不可用时降级 CPU，提示处理时间可能较长
- 移动设备（笔记本电池模式）时提示插电以获得最佳性能

### 4.4 增强历史面板

针对场景二（夜空照片增强），用户可能对同一张原图进行多次不同强度的增强尝试。需提供**增强历史面板**，展示同一张照片的所有增强版本。

**入口**：在照片详情页或右键菜单中 →「增强历史」

**面板内容**：

| 元素 | 说明 |
|---|---|
| 版本列表 | 按时间倒序展示该照片的所有增强版本（缩略图 + 处理时间 + 增强强度值） |
| 当前对比 | 点击任一版本可左右对比该版本与原图 |
| 设为当前 | 用户可选择某个历史版本作为「当前版本」，后续分享/导出以此为基准 |
| 删除版本 | 可删除不再需要的增强版本 |
| 从此版本继续 | 以某个历史版本为起点，调整强度后生成新的增强版本 |

**版本存储**：增强结果与源文件建立关联，存储在派生目录中，不重复占用 OSS 空间（除非用户主动上传）。

## 5. 不做清单

以下场景**不在本次需求范围内**，列入后续规划：

| 场景 | 理由 |
|---|---|
| 深空天体（M42/M31 等）多帧长时间曝光叠加 | 算法复杂度高（需星点对齐 + 消除卫星/飞机轨迹 + 暗场/平场校准），与行星视频叠加有本质不同，独立规划 |
| 窄带通道合成（HOO/SHO 调色盘） | 需要多通道对齐 + 色彩映射引擎，独立规划 |
| HDR 多曝光合成 | 独立规划 |
| 星轨合成 | 独立规划（多图堆栈取最大值，逻辑相对独立） |
| 星野全景拼接 | 拼接算法与月面拼接类似但输入不同（单张照片 vs 视频叠加结果），视月面拼接的完成度再评估 |
| 视频的实时预览叠加 | 仅做离线叠加，不做实时视频预览 |
