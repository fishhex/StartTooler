# Upload Service（上传服务）UI & 交互改进

> 对应 `Views/UploadServerView.axaml`（LAN 上传，265 行）和 `ViewModels/PublicRelayViewModel.cs`（WAN 中转，408 行）。

## LAN 上传（局域网上传）

### 无上传历史
启动服务器后，不知道哪些文件通过手机传过来了，仅有一条 `RecentUploadMessage`（最近一条）。

**方案**: 二维码下方加可滚动日志区域：
```
已上传列表:
  M42_2024-01-15_03 (12.3 MB)  ✓
  NGC7000_2024-01-15_04 (8.7 MB)  ✓
  ...
```
每条显示文件名、大小、时间戳、上传状态（成功/失败），新记录自动滚动到底部。

### 无"复制链接"按钮
展示 `UploadUrl` 和二维码，但无法直接复制。

**方案**: URL 文本框旁加复制按钮（`📋` 图标），点击后按钮文字短暂变为"已复制"。

### 无拖拽上传区域
只支持远程扫码上传，桌面端想传文件还需另开浏览器。

**方案**: 上传页面添加拖放目标区域（Drop Target）：
- 视觉上大虚线框 + 文件图标 + "拖拽文件到此处上传"
- 支持批量拖放
- 上传进度列表

### 端口冲突提示不友好
端口被占用时，`StatusMessage` 显示错误，"启动服务"按钮仍可点击。

**方案**: 端口冲突时：
- 状态消息显示为 `State.Danger` 颜色
- 建议可用端口列表（随机检测 3 个空闲端口）
- "使用随机端口"快捷按钮

### 局域网地址不显示多个 IP
当前 `UploadUrl` 只显示一个地址，多网卡机器可能不准确。

**方案**: 显示所有可用局域网地址的列表，每个带复制按钮。

## WAN 公网中转

### Expander UI 信息密度过高
Expander 展开后是 15+ 个字段的密集表单，无分组、无引导，视觉疲劳。

**方案**: 拆分为两个步骤卡片（Stepper 风格）：
1. **第一步：VPS 连接** — SSH Host / Port / User / Auth / Remote Path
2. **第二步：服务配置** — HTTP Port / TCP Port / Public Host / 部署 & 启动

### 认证方式切换时表单闪烁
`ShowPasswordFields` / `ShowKeyFields` 切换致整个表单布局抖动。

**方案**: Password 和 Key 两个区域高度固定（`MinHeight`），切换时内容替换不导致外部布局变化。

### 部署/启动按钮状态不明确
4 个按钮（保存 / 部署 / 启动 / 停止）一字排开，状态机不清：
- 没保存就部署？没部署就启动？
- 启动中再点启动？

**方案**: 改为线性按钮组（Wizard 风格）：
- 步骤 1：保存 → 成功 → 启用部署
- 步骤 2：部署 → 成功 → 启用启动
- 步骤 3：启动 → 运行中 → 显示停止
- 每步完成后自动启下一步

### 日志区域是单行
`LastLog` 只有一个 `TextBlock` 显示最后一条日志，看不到上下文。

**方案**: 改为多行 `TextBox`（`IsReadOnly="True"`），保留最近 100 行日志，支持滚动查看。添加"清空日志"按钮。

### 无连接状态指示
`RelayStateText` 是纯文本，不够醒目。

**方案**: 状态文字前加彩色圆点指示器（绿=运行中 / 黄=连接中 / 红=断开 / 灰=未启动）。

## 通用

### 上传进度无百分比
`RecentUploadMessage` 只有文件名，无进度信息。

**方案**: 上传中显示进度条 + 百分比 + 已上传/总大小的详情。

### 无 PWA/移动端上传页优化
手机浏览器打开上传页面时，UI 没有移动端适配。

**方案**: `upload.html` 加 `viewport meta` 标签 + 响应式布局，上传按钮做大号触控目标（最小 44x44px）。
