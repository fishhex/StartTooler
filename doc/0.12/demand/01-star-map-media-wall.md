## 1. 需求总览

### 1.1 背景

当前星助浏览照片只有两个入口：

| 入口 | 方式 | 问题 |
|---|---|---|
| 时间轴 | 按拍摄日期折叠，左侧列表 | 线性浏览，无视觉主题感 |
| 标签列表 | 纯文字分组，按标签名排列 | 功能性而非体验性，天文摄影的浪漫感缺失 |

AI 打标白名单包含 18 个星体名称（星云、星系、月亮、土星等），这些标签天然构成了一张"星空地图"——用户拍过的天体在星空中亮起，没拍过的等待探索。

**本需求将标签从文字列表升级为可视化星图，让照片浏览变成"巡天探索"。**

### 1.2 核心价值

| 维度 | 现在 | 将来 |
|---|---|---|
| 浏览方式 | 翻文字列表找标签 | 看星图，点亮的星体 |
| 数据感知 | 不知道哪个天体拍得多 | 星体大小/亮度 = 照片数量，一眼可见 |
| 拍摄引导 | 无 | 灰化的空星体 = 待拍摄清单 |
| 展示交流 | 翻文件夹讲给朋友听 | 打开星图，"你看我拍过的星空" |

### 1.3 一句话概括

**把 18 个星体标签做成艺术化星图，点击星体 → 右侧图墙展示对应媒体。**

## 2. 用户场景

### 场景一：拍完回家，回顾"银河季"成果

> 用户上周末去野外拍了银河、几组星云，偶尔也拍了月亮。今晚回家打开星助。
>
> 切换到星图 Tab，一眼看到「银河」这颗星又大又亮（45张），「星云」中等（18张），「月亮」很小（3张），「土星」是灰的（0张）。
>
> 点击「银河」，图墙跳出 45 张银心特写和广角银河，选几张打开灯箱对比画质。满意，关灯箱。
>
> 再点「星云」，看看那晚拍的 M42 和玫瑰星云哪张噪点少。

### 场景二：规划下次拍摄

> 用户翻看星图，发现「行星」「土星」「木星」都是灰的，想起来去年买的行星相机还没用过。
>
> 决定下个周末去拍木星。

### 场景三：给朋友展示作品集

> 朋友来家里，用户打开星助。不用解释文件夹结构，直接切到星图 Tab。
>
> "你看，这就是我拍过的所有东西"——朋友点「仙女座星系」，看到 12 张深空作品排列在墙上。

## 3. 功能需求

### 3.1 星图入口

| 需求 | 说明 |
|---|---|
| 入口位置 | Gallery 页面左栏 TabControl 新增第三个 Tab「星图」（与「时间」「标签」同级） |
| 切换行为 | 点击「星图」Tab → 左栏切换为星图面板，右侧图墙保留现有照片网格 |
| 选中态同步 | 星图选中星体 → 图墙按标签过滤；切换到「时间」Tab → 星图选中态不丢失 |

### 3.2 星图可视化

#### 3.2.1 星体布局

18 个星体标签按天文属性分 3 个星区，呈现在画布上：

**外环 — 深空天体（蓝紫色调）**
- 星云、星系、星团（3 个）

**中环 — 命名天体（金白色调）**
- 银河、猎户座大星云、仙女座星系、昴星团（4 个）

**内圈 — 太阳系（暖橙色调）**
- 太阳、月亮、行星、彗星、流星、小行星、卫星、土星、木星、火星、金星（11 个）

#### 3.2.2 星体视觉层级

每颗星体由以下元素组成：

| 元素 | 说明 |
|---|---|
| 光晕 | 径向渐变圆，颜色按星区，大小与热度正相关 |
| 核心星点 | 白色/浅色圆点，大小 3–12px = f(照片数量) |
| 标签文字 | 星体名称，中文，居中对齐在星体下方 |
| 照片计数 | 标签文字下方小字，如「42 张」 |

**数据驱动的视觉映射：**

| 照片数 | 状态 | 星点大小 | 光晕 | 文字色 |
|---|---|---|---|---|
| 0 | 未探索 | 3px | 灰色微光，透明度 20% | 灰色 |
| 1–9 | 初探 | 6px | 淡色光晕 | 半亮 |
| 10–29 | 丰富 | 9px | 明显光晕 | 亮色 |
| 30+ | 丰厚 | 12px | 闪耀光晕，微脉冲 | 最亮 |

#### 3.2.3 画布背景

| 元素 | 说明 |
|---|---|
| 底色 | `Bg.Outer`（与 App 深空主题一致） |
| 背景星点 | 随机散布 ~150 个微小星点（0.5–2px），随机透明度 20%–80%，静态 |
| 星云斑块 | 3 个柔和渐变斑块（径向渐变，大半径，透明度 5%–10%），分布在每个星区下方 |

#### 3.2.4 艺术原则

- **艺术化 > 精确性**：不追求天球坐标精度，追求视觉美观与可用性
- **数据驱动**：每颗星的视觉由用户自己的数据决定，千人千面
- **主题统一**：与星助 Deep Space 暗色主题风格一致

### 3.3 交互行为

#### 3.3.1 点击星体

| 行为 | 说明 |
|---|---|
| 选中反馈 | 星体出现脉冲光环（`Accent.Stellar` 色 `#4FC3F7`），该标签高亮 |
| 图墙切换 | 右侧图墙加载该标签对应的媒体文件 |
| 排序 | 默认「时间↓」，可切换「评分↓」，与现有时间/标签 Tab 行为一致 |
| 灯箱 | 点击照片 → 全屏预览，与原 Gallery 行为一致 |
| 多选 | 长按/右键 → 多选模式，支持 AI 打标、上传、删除等批量操作 |

#### 3.3.2 Hover 星体

| 行为 | 说明 |
|---|---|
| 视觉反馈 | 光晕放大 20%，指针变手型 |
| Tooltip | 显示标签名 + 照片数量，如「银河 · 45 张」 |

#### 3.3.3 回到全部

在星图面板顶部提供「全部照片」入口（小文字按钮或图例项），点击后：
- 取消所有星体选中态
- 图墙恢复显示全部照片（同时间 Tab 逻辑）

### 3.4 与现有 Gallery 功能的关系

| 功能 | 星图 Tab 下行为 |
|---|---|
| 排序（时间↓/评分↓） | 与时间/标签 Tab 共享同一个排序下拉框 |
| 灯箱预览 | 完全复用现有 `LightboxWindow` |
| 多选 + 批量操作 | 完全复用现有逻辑 |
| AI 批量打标 | 完全复用，打标完成后星图自动刷新 |
| 上传/下载 | 完全复用 |
| 刷新扫描 | 完全复用，刷新后星图数据更新 |

## 4. 数据需求

### 4.1 数据来源

星图数据完全来自现有 AI 打标结果，不需要新的数据生成流程。

| 数据 | 来源 | 获取方式 |
|---|---|---|
| 每个星体标签的照片数量 | `media_files.tags` 列的 JSON 数组 | 新增 `GetStarTagStatsAsync` 查询 |
| 选中星体对应的照片列表 | 已有 | 复用现有 `GetByTagAsync` |
| 排序 | 已有 | 复用现有 `SortMode` |

### 4.2 新增 Repository 方法

```csharp
// IMediaRepository
/// <summary>
/// 返回 18 个白名单星体标签的媒体数量统计。
/// 只统计 SubjectTagVocabulary 中的标签，排除质量标签和非星体标签。
/// 无媒体的标签返回 Count=0，确保星图上所有 18 个星体都可见。
/// </summary>
Task<IReadOnlyList<TagGroupItem>> GetStarTagStatsAsync(string projectPath, CancellationToken ct);
```

### 4.3 标签白名单

复用 `AITagger.SubjectTagVocabulary`（18 个）：

```
星云, 星系, 星团,                    // 深空天体 (3)
行星, 月亮, 太阳, 彗星, 小行星, 流星, 卫星,  // 太阳系 (7)
猎户座大星云, 仙女座星系, 昴星团, 银河,     // 命名天体 (4)
土星, 木星, 火星, 金星                 // 命名行星 (4)
```

## 6. 空态与边界

| 状态 | 星图表现 | 图墙表现 |
|---|---|---|
| 项目无任何照片 | 所有 18 颗星体灰化 | 现有空态：星座图标 + 「暂无媒体」 |
| 部分星体无照片 | 该星体灰化（大小最小、文字灰色、无光晕），点击无筛选效果 | 保持当前筛选态不变 |
| 全部照片未打标 | 所有星体灰化 | 显示全部照片 + 顶部 banner 提示「AI 打标后将点亮星图」 |
| 打标中 | 星图正常显示，数据基于打标前状态 | 图墙正常显示现有数据 |
| 打标完成 | 刷新星图数据，可能有星体变大/点亮 | 提示「星图已更新」（可选 toast） |
| 数据加载中 | 星图画布低透明度 + 骨架屏效果 | 现有 loading 指示器 |
| 无项目目录 | 与现有 Gallery 行为一致（空态提示） | — |

## 7. 技术约束

| 项 | 约束 |
|---|---|
| 星图绘制 | SkiaSharp 自绘（复用项目现有 SkiaSharp 依赖），不用外部 3D/WebView |
| 性能 | 背景星点缓存为 `SKBitmap`，避免每帧重绘 150 个随机点 |
| 左栏宽度 | 当前硬编码 180px，星图交互需要稍大空间。后续可考虑拖拽调整或适度增加到 220–240px |
| 不修改现有数据模型 | 不新增数据库列、表；数据完全来自现有 `media_files.tags` |

---

## 8. 不做清单

以下内容**不在本次需求范围内**：

| 内容 | 理由 |
|---|---|
| 真实天文星图（赤经赤纬、天球投影） | 艺术化即可，不需要 Stellarium 级别的精确度 |
| 星座连线 | 增加视觉复杂度，与"照片分类入口"核心功能关系弱 |
| 星图缩放/平移 | 当前 18 个星体在 180–240px 宽度内足够展示；后续可评估 |
| 自定义星体位置 | MVP 用固定布局，后续可探索用户自定义排列 |
| 动态星空背景（闪烁/流星动画） | 增加实现复杂度，对核心体验提升有限 |
| WebView 方案 | 坚持 SkiaSharp 自绘，保持技术栈统一 |
| 按星座归类标签 | 当前白名单只有 18 个星体名称，无星座归属信息 |
